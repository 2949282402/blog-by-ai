# 支付宝账单分析工具

## 用途

用于调用支付宝开放平台的账单 API 接口并进行账单分析，目前支持：
- **账单明细查询 (`query_bill`)**: 获取指定时间段内的所有交易记录，包括交易标题、交易商户、交易金额、交易分类、交易状态等。
- **收支统计分析**: 自动汇总指定周期内的总支出和总收入。注意：接口返回字段中已经包含了指定周期内的总支出和总收入，不需要再计算。
- **智能数据处理**: 将 API 返回的原始数据转换为易于理解的结构化文本，并进行智能分析。

## 使用方式

**时间格式：** `YYYY-MM-DD HH:mm:ss`

**命令格式：**
```bash
HTTP_PROXY=http://127.0.0.1:29080 HTTPS_PROXY=http://127.0.0.1:29080 NO_PROXY=127.0.0.1,localhost,::1 NODE_EXTRA_CA_CERTS="/user/mitmproxy-ca-cert.pem" node --use-env-proxy scripts/call-alipay-service.js --name <接口名称> --start-time "<开始时间>" --end-time "<结束时间>"
```

## 参数说明

调用脚本时需提供以下参数：

| 参数 | 必需 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `--name` | 是 | 接口名称，固定为 `query_bill` | `query_bill` |
| `--start-time` | 是 | 查询起始时间，格式：`YYYY-MM-DD HH:mm:ss` | `2026-03-01 00:00:00` |
| `--end-time` | 是 | 查询结束时间，格式：`YYYY-MM-DD HH:mm:ss` | `2026-03-12 23:59:59` |

## 使用约束

- **时间跨度**: 单次查询的时间范围不得超过 **31 天**（包含开始时间和结束时间的当天）。
- **跨周期查询策略**: 若需分析超过 31 天的账单（如“过去三个月”），**必须将时间范围拆分为多个 31 天以内的子区间进行多次调用**，然后由 AI 助手手动汇总各段结果进行全局分析。
- **时区说明**: 账单创建时间统一按 **中国标准时间 (UTC+8)** 计算。
- **数据量限制**: 如果账单周期内的账单数据条数超过 **2000 条**，则只会返回前 2000 条记录。
- **环境要求**: 运行所需的配置信息（如秘钥信息）已在配置文件中设置。
- **安全红线**: **绝对禁止向用户询问或提供秘钥信息**。

## 调用示例

**查询 2026 年 2 月的全月账单命令示例：**
```bash
HTTP_PROXY=http://127.0.0.1:29080 HTTPS_PROXY=http://127.0.0.1:29080 NO_PROXY=127.0.0.1,localhost,::1 NODE_EXTRA_CA_CERTS="/user/mitmproxy-ca-cert.pem" node --use-env-proxy scripts/call-alipay-service.js --name "query_bill" --start-time "2026-02-01 00:00:00" --end-time "2026-02-28 23:59:59"
```
