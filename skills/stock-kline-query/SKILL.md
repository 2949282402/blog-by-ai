# 股票K线数据查询 Skill

## ⚡ 快速参考（每次调用前必确认）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **start** | 当前时间 - 30天 | 开始时间戳（毫秒），**13位数值** |
| **end** | 当前时间 | 结束时间戳（毫秒），**13位数值** |
| **count** | 10 | 数据条数（最大400） |

**【禁止事项】**
- 禁止 start/end/count 为空或 null
- 禁止 period 传枚举值以外的任意值
- 禁止传入字符串类型的时间戳（必须是数值）
- **禁止使用 Date.parse() 计算时间戳**（存在时区解析问题）

**【强制要求】**
- **所有时间戳转换必须使用 `timestamp-convert.js` 脚本**，确保时间戳准确无误

### 1. period 推断（K线周期）

| 用户描述 | period值 | 备注 |
|---------|---------|------|
| 分时/分钟/1分钟 | P_Min1 | |
| 5分钟 | P_Min5 | |

| 周K/周线 | P_Week1 | |
| 月K/月线 | P_Month1 | |
| 年K/年线 | P_Year1 | |

### 2. 时间范围推断（start / end）

| 用户描述 | start计算 | end计算 |
|---------|-----------|---------|
| 最近N天 | 当前-N天 00:00:00 | 当前时间 |
| 最近N周 | 当前-N*7天 | 当前时间 |
| 最近N月 | 当前-N月 | 当前时间 |
| 今天/当日 | 当前日 00:00:00 | 当前时间 |
| 本周 | 本周一 00:00:00 | 当前时间 |
| 本月 | 本月1日 00:00:00 | 当前时间 |
| 本年 | 本年1月1日 00:00:00 | 当前时间 |
| **无明确描述** | **当前-30天（默认值）** | **当前时间（默认值）** |

**时间戳转换方法（必须使用脚本）：**

```bash
# 获取当前时间戳
node timestamp-convert.js -n

# 日期转时间戳
node timestamp-convert.js -t "2026-03-26"
node timestamp-convert.js -t "2026-03-26 23:59:59"

# 生成时间范围（最近N天）
node timestamp-convert.js -r 30

# 时间戳转日期（验证用）
node timestamp-convert.js -d 1743004799999
```

**⚠️ 禁止使用 Date.parse()**：不同环境对日期字符串的解析存在时区差异，会导致时间戳错误！

## 🔧 脚本调用方式

### 前置步骤

**必须先调用 stock-entity-query 获取股票代码**

### Node.js 脚本调用

```bash
HTTP_PROXY=http://127.0.0.1:29080 HTTPS_PROXY=http://127.0.0.1:29080 NO_PROXY=127.0.0.1,localhost,::1 NODE_EXTRA_CA_CERTS="/user/mitmproxy-ca-cert.pem" ALIPAY_KEY_TYPE=PKCS1 node --use-env-proxy ../stock-kline-query/scripts/stock-kline-query.js \
  --config /user/.antConfig/config.json \
  --symbol "000001.SZ" \
  --period "P_Day1" \
  --split "S_Before" \
  --start 1742371200000 \
  --end 1743004799999 \
  --count 10
```

### 参数说明

| 参数 | 必需 | 说明 |
|------|------|------|
| symbol | 是 | 股票代码，格式为"代码.市场"（如000001.SZ、600519.SH） |
| period | 是 | K线周期枚举值：P_Min1/P_Min5/P_Min15/P_Min30/P_Min60/P_Day1/P_Week1/P_Month1/P_Year1 |
| split | 否 | 复权类型：S_Before(前复权)、S_After(后复权)、S_Unsplit(不复权)，默认 S_Before |
| start | **是** | 开始时间戳（毫秒），**13位数值**，默认当前-30天 |
| end | **是** | 结束时间戳（毫秒），**13位数值**，默认当前时间 |
| count | **是** | 数据条数（最大400），默认10 |
| include-start | 否 | 是否包含开始时间，默认 true |
| include-end | 否 | 是否包含结束时间，默认 true |

## ❌ 常见错误（禁止这样做）

### 错误1：传入字符串而不是时间戳数值

```json
// ❌ 错误！传入字符串
{
  "start": "当前时间戳",
  "end": "7天前的时间戳"
}

// ❌ 错误！传入日期字符串
{
  "start": "2026-03-19",
  "end": "2026-03-26"
}

// ✅ 正确！必须是数值类型
{
  "start": 1742371200000,
  "end": 1743004799999
}
```

### 错误2：时间戳精度错误

```json
// ❌ 错误！秒级时间戳（10位）
{
  "start": 1742371200,
  "end": 1743004799
}

// ✅ 正确！毫秒级时间戳（13位）
{
  "start": 1742371200000,
  "end": 1743004799999
}
```

### 错误3：遗漏必须参数

```json
// ❌ 错误！缺少 count 参数
{
  "symbol": "600519.SH",
  "period": "P_Day1",
  "start": 1742371200000,
  "end": 1743004799999
}

// ✅ 正确！所有必须参数都要有值
{
  "symbol": "600519.SH",
  "period": "P_Day1",
  "start": 1742371200000,
  "end": 1743004799999,
  "count": 10
}
```

### 错误4：period 传入错误值

```json
// ❌ 错误！period 不是标准枚举值
{
  "period": "day"      // 错误
  "period": "P_Day"    // 错误
  "period": "DAILY"    // 错误
  "period": "日K"      // 错误
}

// ✅ 正确！必须是标准枚举值
{
  "period": "P_Day1"   // 正确
  "period": "P_Min5"   // 正确
  "period": "P_Week1"  // 正确
}
```

## 辅助工具

该目录下内置了 `timestamp-convert.js` 脚本，**所有时间戳转换必须使用此脚本**。

### timestamp-convert.js 使用指南

```bash
# 获取当前时间戳（最常用）
node timestamp-convert.js -n
# 输出：
#   毫秒时间戳: 1743004799999
#   秒级时间戳: 1743004799
#   本地时间:   2026-03-26 23:59:59

# 生成时间范围（用于K线查询，推荐）
node timestamp-convert.js -r 30
# 输出：
#   开始时间: start = 1740326400000 (2026-02-24 00:00:00)
#   结束时间: end   = 1743004799999 (2026-03-26 23:59:59)
#   K线查询命令示例: ...

# 日期转时间戳
node timestamp-convert.js -t "2026-03-26"
node timestamp-convert.js -t "2026-03-26 09:30:00"
node timestamp-convert.js -t "today"
node timestamp-convert.js -t "yesterday"

# 时间戳转日期（验证用）
node timestamp-convert.js -d 1743004799999
# 输出：
#   毫秒时间戳: 1743004799999
#   本地时间:   2026-03-26 23:59:59
#   ISO 8601:   2026-03-26T15:59:59.999Z
```

### ⚠️ 为什么禁止使用 Date.parse()？

`Date.parse()` 在不同 JavaScript 环境中对日期字符串的解析存在差异：

```javascript
// 不同环境可能得到不同结果
Date.parse('2026-03-26')           // 可能解析为 UTC 或本地时间
Date.parse('2026-03-26 00:00:00')  // 部分浏览器不支持此格式
Date.parse('2026/03/26')           // 格式不统一
```

使用 `timestamp-convert.js` 脚本可以确保时间戳转换的一致性和准确性。
