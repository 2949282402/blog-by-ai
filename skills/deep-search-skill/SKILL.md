# 深度搜索技能

此技能用于指导 AI 执行完整的深度搜索流程，包括多轮搜索和信息整合，最终生成专业的 Markdown 格式搜索报告。

## 核心流程

### 1. 理解需求与规划

- 首先分析总结用户的需求，然后以专业的语气向用户友好地介绍你将做什么，必须保持专业和简洁
- 将大问题拆成 3-6 个可搜索的子问题
- 判断首轮更适合做深度搜索、广度搜索还是数据搜索
- 在开始搜索前，先向用户同步你的理解、子问题和初步计划

若用户需求过宽、过模糊，可先做合理假设并注明。

### 2. 执行搜索

**关键要求：必须使用'mcporter'skill托管的'web-search.web-search'工具进行搜索**

1. **调用 MCP 搜索工具**：使用'web-search.web-search'MCP搜索服务进行网络搜索
   - 搜索关键词应明确、具体
   - 每轮可以并发进行多个搜索（最多3个）
   - 总搜索调用次数默认不超过20次，这里指实际发起的搜索 query 次数，而非搜索轮次
   - 支持深度搜索、广度搜索、数据搜索等不同类型

2. **停止条件与搜索预算**：
   - 不固定搜索轮次，以子任务信息搜索充分作为停止条件
   - 当用户问题的主要子问题已被可信来源覆盖，且已收集足够支撑报告的原始链接、关键事实和必要数据时，应立即停止搜索
   - 当最近一轮新增信息有限、重复结果明显增加时，应停止继续扩展
   - 如果达到20次调用后仍存在明显信息缺口，应先向用户说明缺口和原因，再决定是否继续扩展搜索
   - 每轮搜索后需要向用户汇报收获和下一步方向

3. **搜索类型**：
   - **深度搜索**：深入某个相关子主题
   - **广度搜索**：扩展当前主题，覆盖更多相关领域
   - **数据搜索**：搜索可用于图表展示的数据

### 3. 生成搜索报告

搜索完成后，需要生成专业的搜索报告文档：

**步骤**：

1. **阅读写入提示词**：
   ```
   references/prompts/deep-search-write-file-prompt.md
   ```
   
2. **整理搜索结果**：
   - 汇总所有搜索轮次的信息
   - 提取关键信息和数据
   - 标注信息来源（URL）

3. **生成 Markdown 文档**：
   - 文件名格式：`search_{主题名称}.md`
   - 文件名使用中文，避免特殊字符
   - 使用脚注格式标注信息来源
   - 结构化呈现信息

4. **写入文件到工作目录**：
   - 将生成的 Markdown 文档写入当前工作目录


## 注意事项

### 搜索阶段
- ✅ 必须使用 MCP 搜索服务
- ✅ 每轮搜索后向用户汇报进展
- ✅ 保持专业和简洁的沟通风格
- ✅ 以信息充分度决定是否停止搜索，而不是固定跑满轮次
- ✅ 合理规划并发数量和总搜索预算，默认总调用不超过20次
- ❌ 不要输出完整的搜索报告文本（在搜索阶段）
- ❌ 不要使用 emoji 和复杂 markdown 格式

### 报告生成阶段
- ✅ 必须标注所有信息来源（URL）
- ✅ 使用脚注格式引用来源
- ✅ 文件名使用 `search_` 前缀
- ✅ 文件名使用中文，避免特殊字符
- ✅ 内容结构化、专业化
- ❌ 不要添加欢迎词或多余说明
- ❌ 不要遗漏原始链接

## 关键要点

1. **搜索流程以本文件为准**：按 SKILL.md 中定义的步骤完成需求分析、搜索执行和报告生成
2. **MCP 搜索服务**：通过 MCP 调用搜索功能，而不是其他方式
3. **文件写入**：最终产物是 Markdown 文件，写入到电脑工作目录
4. **用户体验**：在搜索过程中保持与用户的沟通，告知进展和方向
5. **专业性**：生成的报告要结构化、信息全面、来源清晰
