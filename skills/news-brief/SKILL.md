# 新闻与学术简报 Skill

## 功能概述

根据用户输入**智能识别需求类型**（新闻或论文），从权威来源抓取内容，**先提炼核心要点，再输出详细内容**。

| 模式 | 数据来源 | 输出特点 |
|------|---------|---------|
| **📰 新闻模式** | 合法合规的中国大陆媒体 + 优质外文信息源 | 智能排序，优先权威热点 |
| **📚 论文模式** | arXiv、Google Scholar、Semantic Scholar、PubMed | 聚焦研究方法结论 |


## 数据来源

### 📰 新闻来源（开放式多元媒体）

**原则**：不限定特定数据源，覆盖所有合法合规的中国大陆新闻源及优质外文信息源（**无政治风险**）。

#### 来源权威性分级（用于内容排序）

| 等级 | 类型 | 示例 |
|------|------|------|
| **T1** | 权威官方媒体 | 新华社、人民日报、央视、新华网、中国日报 |
| **T2** | 主流商业媒体 | 财新、第一财经、澎湃新闻、界面新闻、新京报 |
| **T3** | 综合门户 | 凤凰网、腾讯新闻、网易新闻、新浪新闻、搜狐新闻 |
| **T4** | 垂直优质媒体 | 36氪、虎嗅、极客公园、钛媒体、创业邦 |
| **T5** | 优质外文媒体 | Reuters、Bloomberg、BBC、The Guardian、Associated Press |

**时效性**：优先展示**最近3天内**发布的新闻，24小时内内容加权。

### 📚 学术来源
1. **arXiv** - 预印本论文
2. **Google Scholar** - 学术搜索
3. **Semantic Scholar** - AI驱动学术搜索
4. **PubMed** - 生物医学文献

**说明**：基于公开可检索资源，无法访问知网等付费数据库。

## 工作流程

### 1. 解析需求与智能识别
- 提取关键词，判断新闻/论文模式
- 新闻类关键词："今天新闻"、"热点"、"时事"
- 论文类关键词："论文"、"学术"、"文献"

### 2. 搜索内容
- 优先使用 `web_search` 工具，若 `web_search` 不可用，使用 `mcporter` 调用 `web_search` 工具进行搜索
- 新闻：使用开放式搜索 `关键词 新闻`，不预设站点限制，覆盖多元媒体
- 论文：`关键词 site:arxiv.org` 或 `关键词 paper`

### 3. 内容排序与筛选

对召回的新闻内容进行智能排序和筛选，优先呈现最新、最相关、最可信的内容。

**排序原则**：
- **时效优先**：越新的新闻越优先展示（24小时内为最新鲜）
- **热度参考**：关注度高、传播广的新闻优先呈现
- **权威佐证**：同等条件下，权威来源（T1/T2级）的内容更可信
- **去重精简**：同一事件的多个报道，保留最新或最完整的一条

**筛选规则**：
- 排除来源不明、无法验证的内容农场
- 排除政治敏感、虚假信息源

### 4. 抓取与生成摘要
- 使用 `web_fetch` 抓取详情
- **新闻**：基于排序后的结果，取Top N生成摘要
- **论文**：提取标题、作者、期刊、摘要
- **去重**：同一事件/论文只保留综合得分最高的一条

### 4. 生成摘要
- **新闻**：提取核心事实（5W1H），删除主观评价，约150字
- **论文**：提取研究背景、方法、核心发现、结论，约150字

### 5. 提炼总结（核心步骤）
**在输出详细内容前，先进行整体总结**：

#### 新闻类总结
- 分析所有新闻的关联性和逻辑脉络
- 提炼3-5个核心要点（每个要点不超过50字）
- 总结应包含：事件核心、主要冲突/进展、影响/趋势

#### 论文类总结
- 分析所有论文的研究主题关联性
- 提炼3-5个核心要点（每个要点不超过50字）
- 总结应包含：研究热点、主要方法、关键发现、发展趋势

**总结原则**：
- 用户30秒内可读完
- 优先呈现最重要的信息
- 避免重复，合并相似内容
- 客观中立，不加入主观判断

### 6. 关键词推荐与气泡引导（强制步骤）

**⚠️ 必须执行，不可跳过**
在完成总结后，基于当前搜索的关键词和检索到的内容，**必须生成2-3个相关关键词推荐**。

#### 生成规则
- **数量要求**：必须生成2-3个关键词，不能为0，也不能超过3个
- **强制生成**：即使搜索结果较少，也必须从内容中提取相关关键词
- **无内容时的处理**：如果确实无法从内容中提取，基于原关键词进行合理联想扩展
- **时效性**：优先推荐当前热门或趋势性的相关主题
- **多样性**：确保推荐的关键词与原关键词有区分度，避免高度重复


#### 推荐示例
- 用户搜索"人工智能" → 推荐：["大模型", "自动驾驶"]
- 用户搜索"深度学习" → 推荐：["Transformer", "计算机视觉"]
- 用户搜索"新能源汽车" → 推荐：["锂电池", "充电桩"]


在完成简报输出后，**必须调用 RENDER_MINI_CARD 工具**返回可点击的关键词按钮。

**⚠️ 重要说明**：
- `RENDER_MINI_CARD` 是一个**工具**，不是文本指令，不能直接输出。
- 不需要在调用工具前输出关键词推荐文字，工具会自行渲染展示
- 必须通过**工具调用**的方式传入 JSON 参数，单纯输出 JSON 文本不会生成气泡
- JSON 参数是工具的入参，**绝对不要在回复中直接展示给用户**

**调用方式**：
调用 RENDER_MINI_CARD 工具，传入以下格式的参数：

```json
{
  "cardType": "actions",
  "cardData": {
    "actions": [
      {"text": "关键词1", "link": "sendchat://关键词1"},
      {"text": "关键词2", "link": "sendchat://关键词2"}，
      {"text": "关键词3", "link": "sendchat://关键词3"}
    ]
  }
```

## 重要约束（必须遵守）

### 来源标注
- **每条新闻必须标注具体媒体名称**，未标注的不输出
- **每篇论文必须标注作者、期刊、发表时间**，未标注的不输出

### 原文链接（核心原则：真实、准确、可访问）
**⚠️ 核心原则：链接必须真实、准确、可访问**
1. **禁止虚构链接**：严禁使用占位符链接（如 `https://www.xinhuanet.com/...`、`<URL>`、空链接等）
2. **链接来源**：链接必须来自搜索结果中的真实URL，不得自行构造
3. **完整URL**：链接必须是完整的、可直接访问的URL地址
4. **一一对应**：每条新闻/论文只能有一个原文链接，且必须能跳转到对应页面
5. **无链接则不输出**：如果无法获取真实链接，该条新闻/论文不得出现在简报中
6. **原本链接不得直接输出** 原本链接必须使用markdown的语法，如[原文链接](URL)，不能直接对用户展示`https://www.xinhuanet.com/`
   **获取链接的正确流程**：
1. 使用 web_search 工具搜索时，从返回结果中提取每条新闻/论文的真实URL
2. 使用 web_fetch 工具抓取页面时，确认URL正确无误
3. 在输出中直接使用搜索结果中的原始URL，不要做任何修改或简化

### 内容质量
- 新闻客观中立，论文学术准确
- 摘要约150字
- 同一事件/论文只保留一条

### 数量控制
- 详细新闻/论文**不超过3条**
- "共检索到 N 条"中的 **N 最大为3**

### 关键词与工具调用（强制）
- 必须生成 **2-3个关键词**
- 必须调用 **RENDER_MINI_CARD 工具**
- 直接调用工具，不要先输出文字

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 工具不可用 | 告知用户"抱歉，当前环境无法获取实时新闻/论文" |
| 搜索无结果 | 建议更换关键词或扩大搜索范围 |
| 无法获取真实链接 | 该条内容不输出，说明"已跳过无法获取链接的内容" |
| 无法确定意图 | 询问"您想了解新闻动态，还是查找学术论文？" |
